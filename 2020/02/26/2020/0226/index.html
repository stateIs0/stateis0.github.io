1 管理哲学

到底是要“监督”还是“自驱”？如果是后者的话，则需要一个强大的愿景作为驱动.

2 细节

1 分享：每周让每个人分享自己的工作感悟和心得，然后发精心挑选的奖品；

2 目标和管理：

  • OKR
  • 计划指定和执行跟踪

OKR：

相比较 KPI 来说：
1 更强调由团队成员共同指定目标，容易达成共识，且目标导向一致；
2 能让团队成员知道自己工作内容在整个大局的意义，这样，能促进人与人的协作，让团队具有整体性
3 透明，每个人都能看到其他人的 OKR，知道其他人在做什么

计划指定和执行跟踪

每个具体的项目计划需要透明，每个负责人都为自己的项目建立一个全局的大盘，也可以称为地图。
地图上有【资源的链接、负责人、风险点、进度】，这个对于管理者来说，至关重要；

3 工具

  • GitHub 代码托管
  • Zoom 在线会议
  • Slack 即时通讯（电话）
  • 企业微信
  • 在线文档
  • 邮件日历
  • wiki Confluence（内部文档）
  • Jira bug 跟踪
  • Trello 看板，记录重要客户和事件备忘
  • Jenkins CI

例子：如果开发一个小功能

  1. 使用在线文档草拟描述，并分享同事
  2. 文档沟通定稿，同步 wiki 和 gitHub
  3. 拆分子项目，通过 Jira 分配，完成后提交到 Github
  4. 持续集成

注意点：工具，不仅仅是使用现成的，工程师也可以开发工具来提升效率；能交给机器做的，就尽量不要让人来做，自动化是至关重要的；

4 远程最大的挑战：沟通

  1. 沟通必不可少，沟通的品质决定了做事情的本质；

远程的状态下，沟通会更多，更细致，并不会变少；只是形式不再是面对面的会议；

沟通最重要的意义：信息拉平

大白话：大家都互相知道自己该干嘛，团队正在干嘛以及该干嘛；

要做到这点，还是需要透明；

即使在远程，会议仍然是信息拉平的重要方式。Zoom 视频会议工具很优秀；

会议分2种：

  • 例会（通常以周为单位，管理者可以知道方向是否正确，进度是否正常）
  • 具体业务的沟通会

注意：

  • 例会上，管理者不要轻易做决策，应当会后思考后，再进行决策；
  • 善用日历，让别人可见，目的是信息拉平
  • 会议前发议程，会议后发笔记，并记录 wiki；
  • 少开大会，长会，两个 pizza 原则（两张 pizz 就能喂饱团队）

团队周报和个人周报，都记录在 wiki 里，并且标注索引；

5 远程办公的自我管理

家里或者咖啡厅

需要克服孤独感，需要自律精神，克服倦怠感；使用个人时间管理工具，例如番茄钟，日历等工具；

作为管理者，任务要拆解的足够细，在前期，需要和远程同事，反复确认是否达成一致；

总结

  1. 自驱为主
  2. OKR 制定计划，透明，公开
  3. 地图跟踪任务，周报总结任务，例会调整任务
  4. 沟通是重点：使用工具进行沟通，例如电话，视频会议，在线文档（文档驱动很重要）
  5. 人：能独当一面的人、沟通能力要强、能自我管理和自驱动

到底是花时间“教育不好的人”，还是“花时间精力找更好的人” ？ 当然是后者。

制度的建立，要基于好人更多的原则来制定。

一个普通员工的一天

  1. 看下 OKR 上自己的任务，再看看地图上的大盘，看看别人做了啥，自己到哪一步了；
  2. 根据项目进度和 leader 的分配，开始做今天的任务，用小番茄定个任务；
  3. 如果遇到困难，可以和同事或者 leader 沟通；
  4. 每天结束，更新自己在地图上的进度，写上今天的计划，优先级、问题，总结摘要；
  5. 每周的周报，工作心得和感悟
